# 飛買家銷售資料彙整工具

自動彙整電商平台的 Excel 銷售明細，依 SKU 加總銷售量、銷售額，並整合庫存、成本等資料，匯出彙整後的報表。

網站：https://zeabur-test1.zeabur.app/

---

## 輸入檔案說明

以下各類檔案均為 `.xlsx` / `.xls` 格式，可單獨或任意組合上傳。

### 銷售明細（可多份，每份對應一個月份）

| 必要欄位 | 說明 |
|----------|------|
| `SKU no` | 商品 SKU 編號 |
| `SKU title` | 商品名稱 |
| `Volume` | 銷售量 |
| `Amount` | 銷售額 |

> 若 Excel 內有多個工作表，程式自動讀取名為 `details` 的工作表；只有一個工作表時直接讀取。

---

### 採購未交量（在途庫存來源）

| 必要欄位 | 說明 |
|----------|------|
| `品    號` | 商品品號（資料從第 4 列起） |
| `品   名` | 商品名稱 |
| `未交數量` | 未交貨數量 |
| `交貨庫` | 篩選條件 |

**篩選邏輯：**
- 僅保留 `交貨庫 = 華膳-IT` 的列
- 僅保留品號結尾為 `A` 的品項，取前 5 碼作為 SKU No.（例：`00123A` → `00123`）

---

### 機上量（需同時上傳一般航線與串飛航線）

| 必要欄位 | 說明 |
|----------|------|
| `SKU No.` | 商品 SKU 編號（資料從第 2 列起） |
| `數量` | 裝載數量 |
| `DESCRIPTION` | 商品描述 |

**計算邏輯：**
```
機上量 = (一般航線裝載數量 × 41) + (串飛航線裝載數量 × 10)
```

---

### 期末存量

| 必要欄位 | 說明 |
|----------|------|
| `料號` | 商品料號（資料從第 6 列起） |
| `品名` | 商品名稱 |
| `期末存量` | 期末庫存數量 |

---

### 進貨明細（本月進貨來源）

| 必要欄位 | 說明 |
|----------|------|
| `品號` | 商品品號（資料從第 4 列起） |
| `品名` | 商品名稱 |
| `驗收數量` | 驗收入庫數量 |
| `庫別` | 篩選條件 |

**篩選邏輯：**
- 僅保留 `庫別 = 華膳-IT` 的列
- 僅保留品號結尾為 `A` 的品項，取前 5 碼作為 SKU No.

---

### 品號價格資料（商品成本來源）

| 必要欄位 | 說明 |
|----------|------|
| `品號` | 商品品號（資料從第 3 列起） |
| `品名` | 商品名稱 |
| `幣別名稱` | 採購幣別 |
| `採購單價` | 商品採購單價 |
| `核價日` | 核價日期 |

**計算邏輯：**
- 僅保留品號結尾為 `A` 的品項，取前 5 碼作為 SKU No.
- 同一品號若有多筆記錄，取**最新核價日**那一筆
- 若幣別非台幣，上傳時系統會提示輸入匯率，並自動換算：
```
TWD成本 = 採購單價 × 匯率（四捨五入至整數）
```

---

### 採購大表（觸發補貨計算）

上傳此檔案後，系統會額外執行補貨計算，輸出虎航庫存計算表。

| 必要欄位 | 說明 |
|----------|------|
| `SKU No.` | 商品 SKU 編號（資料從第 3 列起） |
| `DESCRIPTION` | 商品描述 |
| `規劃性下架` | 是否規劃下架（有值則補貨量強制為 0） |
| `成箱規定` | 每箱數量（預設 5，補貨量取此倍數的最小整數倍） |
| `lead time` | 備貨所需月數 |

---

## 輸出報表欄位說明

### 基本報表（未上傳採購大表）

輸出單一工作表「報表」，欄位如下：

| 欄位 | 計算方式 / 來源 |
|------|----------------|
| `SKU No.` | 正規化後的 SKU 編號（統一補零至 5 位） |
| `品名` | 優先採用：銷售明細 > 在途庫存 > 機上量 > 期末存量 > 本月進貨 > 商品成本 |
| `TWD成本` | 採購單價 × 匯率（台幣直接使用原價） |
| `機上量` | 一般航線 × 41 + 串飛航線 × 10，同 SKU 加總 |
| `{月份}銷售量` | 當月各筆銷售量加總（可多個月份，各自一欄） |
| `{月份}銷售額` | 當月各筆銷售額加總（可多個月份，各自一欄） |
| `期末存量` | 直接來自期末存量檔案 |
| `本月進貨` | 驗收數量加總（庫別=華膳-IT，品號尾碼A） |
| `在途庫存` | 未交數量加總（交貨庫=華膳-IT，品號尾碼A） |

> 所有來源以 SKU No. 為 key 做 outer join，缺少的欄位留空（NaN）。

---

### 補貨計算報表（上傳採購大表後）

在基本報表欄位之外，額外新增以下欄位：

| 欄位 | 計算公式 |
|------|----------|
| `平均月銷量` | 所有月份銷售量的簡單平均（四捨五入） |
| `本月銷貨({月份})` | 最新月份的銷售量 |
| `需求量(1.5)` | `機上量 + MAX(平均月銷量, 本月銷貨) × 1.5` |
| `補貨量(以1.5個月)` | `MAX(需求量(1.5) − (期末存量 − 本月銷貨) − 本月進貨 − 在途庫存, 0)`，無成箱規定則直接輸出，有則無條件進位至成箱倍數 |
| `採購金額(1.5)` | `補貨量(以1.5個月) × TWD成本` |
| `需求量_lead_time` | `機上量 + MAX(平均月銷量, 本月銷貨) × lead time` |
| `補貨量(以lead time)` | `MAX(需求量_lead_time − (期末存量 − 本月銷貨) − 本月進貨 − 在途庫存, 0)`，同上取成箱倍數 |
| `採購金額(lead time)` | `補貨量(以lead time) × TWD成本` |
| `追加數量` | `補貨量(以lead time) − 補貨量(以1.5個月)` |

> 若 `規劃性下架` 欄位有非空值，該品項的所有補貨量強制為 0。

**補貨量邏輯說明：**

「現有可用庫存」= 期末存量 − 本月銷貨 + 本月進貨 + 在途庫存

補貨量 = 需求量 − 現有可用庫存，結果小於 0 時取 0，再依成箱規定無條件進位。

## 輸出檔名規則

| 情境 | 檔名 |
|------|------|
| 有上傳採購大表 | `虎航庫存計算表({最新月份}).xlsx` |
| 有銷售明細、無採購大表 | `TTW sales summary {起月}-{迄月}.xlsx` |
| 僅庫存類資料、無銷售明細 | `TTW 庫存表.xlsx` |
